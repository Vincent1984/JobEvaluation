# 分类标签管理功能 - 用户指南

## 功能概述

分类标签管理功能允许您为第三层级职位分类添加标签，这些标签将在岗位评估时作为重要的参考因素，帮助系统更准确地判断岗位价值和核心岗位属性。

## 快速开始

### 1. 进入职位分类管理页面

在左侧导航栏选择 **"🗂️ 职位分类管理"**

### 2. 找到第三层级分类

在分类树中展开到第三层级分类（例如：技术类 → 研发 → 后端工程师）

### 3. 查看标签徽章

第三层级分类名称后会显示标签数量徽章：
- 📄 后端工程师 (L3) 🏷️ 2  ← 表示有2个标签

## 功能详解

### 一、查看标签列表

1. 展开第三层级分类
2. 在分类信息下方找到 **"🏷️ 分类标签管理"** 区域
3. 点击 **"📋 查看所有标签"** 展开标签列表

**标签信息包括：**
- 标签名称（粗体显示）
- 标签类型（括号中显示）
- 标签描述（斜体显示）

**示例：**
```
高战略重要性 (战略重要性)
该岗位对公司战略目标实现具有重要影响
```

### 二、添加新标签

1. 在标签管理区域找到 **"➕ 添加新标签"** 表单
2. 填写以下信息：

   **标签名称*** （必填）
   - 输入标签的名称
   - 示例：高战略重要性、核心技术岗位

   **标签类型*** （必填）
   - 从下拉菜单选择一种类型：
     - 战略重要性：岗位对战略目标的影响
     - 业务价值：岗位对业务的直接贡献
     - 技能稀缺性：所需技能的稀缺程度
     - 市场竞争度：市场上的竞争激烈程度
     - 发展潜力：岗位的发展空间
     - 风险等级：岗位的风险程度

   **标签描述*** （必填）
   - 详细说明该标签的含义
   - 说明该标签如何影响岗位评估
   - 示例：该岗位对公司战略目标实现具有重要影响，在综合评估时会提升岗位价值评级

3. 点击 **"✅ 添加标签"** 按钮
4. 系统会显示成功提示并刷新页面

### 三、编辑标签

1. 在标签列表中找到要编辑的标签
2. 点击标签右侧的 **"✏️"** 按钮
3. 右侧边栏会显示编辑表单
4. 修改标签名称、类型或描述
5. 点击 **"💾 保存"** 保存修改
6. 或点击 **"❌ 取消"** 放弃修改

**注意：**
- 编辑标签时，所有字段都是必填的
- 标签类型只能从预定义的6种类型中选择
- 修改会立即生效，影响后续的岗位评估

### 四、删除标签

1. 在标签列表中找到要删除的标签
2. 点击标签右侧的 **"🗑️"** 按钮
3. 系统会立即删除标签并刷新页面

**警告：**
- 删除操作不可撤销
- 删除标签后，已使用该标签的评估结果不会受影响
- 但新的评估将不再考虑该标签

## 标签类型说明

### 1. 战略重要性
**用途：** 标识岗位对公司战略目标的影响程度

**示例标签：**
- 高战略重要性：核心战略岗位
- 中战略重要性：支持战略实施
- 低战略重要性：辅助性岗位

**评估影响：** 高战略重要性的岗位更可能被判定为核心岗位，企业价值评级更高

### 2. 业务价值
**用途：** 标识岗位对业务的直接贡献

**示例标签：**
- 高业务价值：直接创造营收
- 中业务价值：支持业务运营
- 低业务价值：间接支持

**评估影响：** 高业务价值的岗位在综合评估中得分更高

### 3. 技能稀缺性
**用途：** 标识岗位所需技能的稀缺程度

**示例标签：**
- 技能稀缺：市场上难以招聘
- 技能常见：市场供应充足

**评估影响：** 技能稀缺的岗位更可能被判定为核心岗位

### 4. 市场竞争度
**用途：** 标识岗位在人才市场的竞争激烈程度

**示例标签：**
- 高竞争度：多家企业争抢
- 中竞争度：正常竞争
- 低竞争度：竞争较小

**评估影响：** 高竞争度的岗位在薪酬和福利评估中权重更高

### 5. 发展潜力
**用途：** 标识岗位的成长空间和发展前景

**示例标签：**
- 高发展潜力：晋升空间大
- 中发展潜力：稳定发展
- 低发展潜力：发展受限

**评估影响：** 高发展潜力的岗位在候选人匹配时更具吸引力

### 6. 风险等级
**用途：** 标识岗位的风险程度

**示例标签：**
- 高风险：关键岗位，离职影响大
- 中风险：正常风险
- 低风险：可替代性强

**评估影响：** 高风险岗位需要更严格的招聘标准和更好的留任措施

## 最佳实践

### 1. 标签数量建议
- 每个第三层级分类建议添加 **3-10个标签**
- 太少：评估参考不足
- 太多：增加复杂度，影响评估效率

### 2. 标签命名规范
- 使用清晰、简洁的名称
- 避免使用模糊或主观的词汇
- 保持命名风格一致

**好的示例：**
- ✅ 高战略重要性
- ✅ 核心技术岗位
- ✅ 市场稀缺人才

**不好的示例：**
- ❌ 很重要
- ❌ 好岗位
- ❌ 必须要的

### 3. 标签描述要点
- 明确说明标签的含义
- 说明标签对评估的具体影响
- 提供判断依据或标准

**好的描述示例：**
```
该岗位对公司战略目标实现具有重要影响，在综合评估时会：
1. 提升岗位价值评级（高价值）
2. 增加核心岗位判定权重
3. 影响薪酬定位和福利配置
```

### 4. 标签维护
- 定期审查标签的准确性
- 根据业务变化更新标签
- 删除不再适用的标签
- 保持标签体系的一致性

## 常见问题

### Q1: 为什么只有第三层级分类可以添加标签？
**A:** 第三层级是最具体的分类（如"后端工程师"），标签在这个层级最有意义。上层分类（如"技术类"）过于宽泛，不适合添加具体标签。

### Q2: 标签会影响哪些评估结果？
**A:** 标签会影响：
- 综合质量分数
- 企业价值评级（高/中/低价值）
- 核心岗位判断（是/否）
- 各维度贡献度分析

### Q3: 可以为同一个分类添加多个相同类型的标签吗？
**A:** 可以。例如，可以同时添加"高战略重要性"和"中战略重要性"两个标签，但建议每种类型只添加一个最合适的标签，避免混淆。

### Q4: 删除标签后，之前的评估结果会改变吗？
**A:** 不会。已完成的评估结果会保留原有的标签信息。但新的评估将不再考虑被删除的标签。

### Q5: 如何知道标签是否设置合理？
**A:** 可以通过以下方式验证：
1. 对同一岗位进行多次评估，观察结果是否符合预期
2. 比较不同分类的评估结果，检查差异是否合理
3. 查看评估报告中的"维度贡献度"，了解标签的实际影响

## 使用场景示例

### 场景1：创建技术岗位分类标签

**分类：** 技术类 → 研发 → 后端工程师

**添加标签：**

1. **高战略重要性** (战略重要性)
   - 描述：后端工程师是公司核心技术团队的重要组成部分，直接影响产品质量和用户体验

2. **高业务价值** (业务价值)
   - 描述：负责核心业务系统开发，直接支撑公司主营业务运营

3. **技能稀缺** (技能稀缺性)
   - 描述：需要3年以上Python开发经验和微服务架构能力，市场上符合条件的候选人较少

4. **高竞争度** (市场竞争度)
   - 描述：互联网行业对后端工程师需求旺盛，多家企业竞争优秀人才

### 场景2：创建管理岗位分类标签

**分类：** 管理类 → 部门管理 → 技术总监

**添加标签：**

1. **极高战略重要性** (战略重要性)
   - 描述：技术总监负责技术战略规划和团队管理，对公司技术发展方向有决定性影响

2. **高业务价值** (业务价值)
   - 描述：通过技术创新和团队建设，提升整体研发效率和产品竞争力

3. **高发展潜力** (发展潜力)
   - 描述：可晋升为CTO或VP，职业发展路径清晰

4. **高风险** (风险等级)
   - 描述：关键管理岗位，离职会对团队稳定性和项目进度产生重大影响
